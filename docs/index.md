# Curtin ML Group — Handbook

Welcome. This is the one place that explains **how we work** — the tools, the workflow, and what's expected. If you're new, start with [Getting started](getting-started.md).

We do machine learning and AI for structural engineering. Our work spans three funded projects that fit together as **one infrastructure-monitoring stack**:

| Layer | Project | What it does |
|---|---|---|
| Physics | **DoT** | Vehicle–barrier collision simulation |
| Perception | **Linkage** | Bridge digital-twin damage detection & quantification |
| Reasoning | **DECRA** | Agentic AI that closes the loop between them |

Your workstream is one piece of that stack — this handbook is how the pieces stay coordinated.

## The idea in one sentence
Everything about your work lives on GitHub — code, tasks, experiments, decisions — so it's visible, reproducible, and easy to pick up. Large files (datasets, 3D, checkpoints) live in the right specialist tool, with a pointer from GitHub.
