# Using AI (Claude Code)

We use AI tools — including [Claude Code](https://claude.com/claude-code) — as part of normal work. Here's the deal.

## Allowed, and encouraged
- Writing, refactoring, and debugging code.
- Drafting and editing text.
- Exploring literature and explaining concepts.

## Your responsibilities
- **You must understand and be able to defend every line** you commit. AI is a tool, not an author.
- **Flag AI-generated results** as such when you report them. Your supervisor is the final arbiter of what ships.
- **Never paste unpublished partner or DoT data** into external tools without checking the data agreement first.

## The `CLAUDE.md` file
Each repo has a `CLAUDE.md` that tells Claude Code the project's conventions — stack, where data goes, coding style. Keep it current; it steers every AI session in your repo toward our standards.

## Supervisor commands
Your supervisor uses Claude Code commands like `/weekly-digest` (a cross-repo summary) and `/review-pr` (a first-pass PR review). Clear issues, clean PRs, and good W&B logging make these work well — which means faster feedback for you.
